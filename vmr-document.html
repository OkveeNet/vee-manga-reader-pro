Vee's manga reader pro

การติดตั้ง

  1. แตกไฟล์ออกมาจากแพ็ค แล้วนำทั้งหมดไปไว้ในตำแหน่งใดๆของเว็บ หรือ อัปโหลดขึ้นเซอร์ฟเวอร์ทาง ftp
  2. กำหนดตำแหน่งของเว็บอ่านการ์ตูน เปิดไฟล์ .htaccess แล้วแก้ไข RewriteBase เช่น ต้องการไว้ใน http://localhost ให้ระบุ / หรือต้องการไว้ใน http://localhost/vmr ให้ระบุ /vmr/
  3. กำหนด permission ของ folder client/manga, client/avatar, client/upload ให้เป็น write ได้ หรือ 0777 หรือ 777
  4. แก้ไขอีเมลผู้ส่งและ smtp server ใน application/config/email.php

ติดตั้งฐานข้อมูล

  1. ทำการสร้างฐานข้อมูลขึ้นมา จากนั้นจดชื่อฐานข้อมูลไว้
  2. ทำการ import ไฟล์ vmr2 install.sql เข้าไปในฐานข้อมูล โดยใช้ phpmyadmin
  3. เปิดไฟล์ application/config/database.php
  4. $db['default']['username'] ให้กำหนด username ของฐานข้อมูลของคุณ ตัวอย่าง $db['default']['username'] = 'root';
  5. $db['default']['password'] ให้กำหนด password ของฐานข้อมูลของคุณ
  6. $db['default']['database'] ให้กำหนดชื่อฐานข้อมูล

ทดสอบการทำงาน

  1. ทดลองเรียกดูเว็บ ตามตำแหน่งที่ติดตั้งไว้ในข้อ 2. เช่น http://localhost/vmr โดยเปลี่ยน localhost เป็นชื่อโดเมนที่ติดตั้งอยู่
  2. หากพบแบนเนอร์การ์ตูน และไม่มีแสดง error ใดๆ แสดงว่าติดตั้งถูกต้องแล้ว
  3. ทดลองเรียกหน้าผู้ดูแล โดยเพิ่ม /index.php/site-admin เข้าไปจาก url หลัก เช่น http://localhost/vmr/index.php/site-admin
  4. เมื่อเข้าใช้ครั้งแรกจะต้องกรอก username และ password โดยกรอกเป็น admin และ pass ตามลำดับ
  5. หาก log in หน้าผู้ดูแลสำเร็จ จะเข้าหน้าแรกของผู้ดูแล หากไม่มี error ใดๆ แสดงว่าติดตั้งถูกต้องแล้ว
  6. ยินดีด้วย คุณสามารถเปลี่ยนรหัสผ่านแล้วเริ่มใช้งานได้เลย

การปรับแต่งเพื่อเอา index.php ออก

  1. เปิดไฟล์ .htaccess
  2. แก้ไข RewriteBase /vmr-pro2/ เป็นตำแหน่งของเว็บที่คุณติดตั้ง โดยหากอยู่ที่ root web ให้กำหนดเป็น /
  3. เปิดไฟล์ application/config/config.php แก้ไข $config['index_page'] = 'index.php'; เป็น $config['index_page'] = '';
  4. ทดสอบโดยเรียก http://localhost ลองคลิกที่หน้าสารบัญ URL ควรจะได้เป็น http://localhost/directory และไม่มี error ใดๆ

การใช้งานหน้าผู้ดูแล

  1. การปรับแต่งระบบ คุณสามารถเปลี่ยนชื่อเว็บไซต์และตั้งค่าต่างๆได้จากตรงนี้ ในหน้าหนี้มีคำอธิบายไว้อยู่แล้ว
  2. ประเภทการ์ตูน
    1. การเพิ่มประเภท ทำได้โดยไปที่เมนูประเภท แล้วเลือกที่เพิ่มประเภท
    2. กรอกชื่อประเภท และ URI โดย URI คือตำแหน่งของหน้าแสดงประเภทการ์ตูน ซึ่งไม่รวม http (โปรดอ่านคำอธิบายและดูตัวอย่างในหน้าเพิ่มประเภท)
    3. เปิดใช้งานก็คือการเปิดให้ผู้คนทั่วไปเข้าถึงได้ หากปิดอยู่ผู้ใช้ทั่วไปจะไม่สามารเข้าถึงได้เลย
    4. การแก้ไขหรือลบประเภท ทำได้ภายในหน้าเดียวกัน
  3. การ์ตูน
    1. การเพิ่มประเภทการ์ตูน ไปที่เมนูการ์ตูน แล้วเลือกที่เพิ่มการ์ตูน ในหน้านี้จะมีคำอธิบายที่จำเป็นครบถ้วนอยู่แล้ว โปรดทำความเข้าใจก่อนกดส่งข้อมูล
      1. ตำแหน่ง URI สำหรับการ์ตูนนั้น จะไม่สามารถแก้ไขได้ เพราะมันจะสัมพันธ์ถึงตำแหน่งการเก็บไฟล์ภาพด้วย ดังนั้น เมื่อพิมพ์ไปแล้วโปรดคลิกในช่องอื่นใดๆเพื่อดูการตรวจสอบที่ทำงานผ่าน server ว่าผลลัพธ์เป็นที่พอใจหรือไม่ คุณสามารถแก้ไขได้จนกว่าจะพอใจ
    2. การแก้ไขการ์ตูน คลิกที่เมนูการ์ตูน แล้วคลิกแก้ไขในบรรทัดของการ์ตูนที่ต้องการแก้ไข
      1. คุณจะไม่สามารถแก้ไข URI ได้ และเราไม่แนะนำให้คุณทำผ่าน ftp นอกจากคุณแน่ใจว่าคุณรู้วิธีแก้มันใน sql ด้วย
      2. เมื่อแก้ไขเสร็จแล้วกดปุ่ม ส่งข้อมูล
    3. การลบการ์ตูน
      1. ติ๊กในช่องสี่เหลี่ยม ที่หน้าชื่อการ์ตูนที่ต้องการลบ แล้วเลือกรายการลบที่ด้านล่าง จากนั้นกดปุ่ม ส่งข้อมูล
  4. ตอนของการ์ตูน กดที่เมนูการ์ตูน จากนั้นตรงการ์ตูนที่ต้องการเพิ่มตอน คลิกที่ตอน จะแสดงตอนต่างๆของการ์ตูนเรื่องนั้น
    1. การเพิ่มตอน
      1. คลิกที่ปุ่มเพิ่ม
      2. กรอกลำดับตอน โดยควรจะอ้างอิงจากเลขลำดับตอนที่มาจากผู้เขียน ตรงนี้สำคัญมาก หากกรอกลำดับผิด การแสดงผลจะผิดไปด้วย
      3. ในช่อง URI ควรกรอกแล้วคลิกที่อื่นๆเพื่อรอผลการตรวจสอบจาก server เสียก่อน จากนั้นค่อยแก้ไขจนพอใจ.
      4. URI นั้นไม่สามารถแก้ไขได้ เพราะจะสัมพันธ์กับตำแหน่งเก็บไฟล์ภาพ ดังนั้นโปรดรอการตรวจสอบและแก้ไขตรงขั้นนี้จนกว่าจะพอใจ
      5. การเปิดใช้งานหมายถึงผู้ใช้ทั่วไปจะสามารถเข้าถึงได้
      6. ไฟล์ภาพการ์ตูนในตอน โปรดอ่านคำอธิบายจากในหน้าเว็บโดยละเอียด
      7. กดปุ่มส่งข้อมูลเพื่อเพิ่มตอน
    2. การแก้ไขตอน
      1. จากรายชื่อตอน ให้กดปุ่มแก้ไข
      2. URI และไฟล์ภาพ ไม่สามารถแก้ไขได้
      3. การเปิดใช้งานหมายถึงผู้ใช้ทั่วไปจะสามารถเข้าถึงได้
    3. การลบตอน
      1. ติ๊กถูกในช่องสี่เหลี่ยมตรงหน้าตอนที่ต้องการลบ
      2. เลือกตัวเลือก ลบ จากนั้นกดปุ่ม ส่งข้อมูล
  5. ผู้ใช้ ระบบนี้สามารถเพิ่มผู้ดูแลได้หลายระดับ คลิกที่เมนูผู้ใช้ เพื่อดูรายชื่อผู้ใช้
    1. การเพิ่มผู้ใช้
      1. กดที่ปุ่มเพิ่ม
      2. กรอกชื่อผู้ใช้ (ใช้ตอน log in) รหัสผ่าน อีเมล เลือกระดับ และสถานะ จากนั้นกดปุ่ม ส่งข้อมูล
    2. การแก้ไขผู้ใช้
      1. ในบรรทัดรายชื่อผู้ใช้ที่ต้องการแก้ไข ให้กดปุ่มแก้ไข
      2. ชื่อผู้ใช้ไม่สามารถแก้ไขได้ แก้ไขโดยหลักการเดียวกันกับการเพิ่มผู้ใช้ โดยหากต้องการเปลี่ยนรหัสผ่านให้กรอกลงไป หากไม่เปลี่ยนให้ปล่อยว่างไว้
      3. กดปุ่มส่งข้อมูลเพื่อบันทึก
    3. การแก้ไขข้อมูลตนเอง ไปที่เมนูผู้ใช้ แล้วคลิกที่ แก้ไขข้อมูลของฉัน
    4. การลบผู้ใช้
      1. ติ๊กถูกในช่องสี่เหลี่ยมตรงหน้าผู้ใช้ที่ต้องการลบ เลือกตัวเลือกลบ แล้วกด ส่งข้อมูล
    5. ผู้ใช้ที่ระดับต่ำกว่าผู้ใช้ที่ถูกเลือก จะไม่สามารถดำเนินการใดๆได้ รวมทั้งไม่สามารถตั้งผู้ใช้ที่ระดับสูงกว่าตนเองได้
    6. ผู้ใช้รายแรก ID 1 คือเจ้าของเเว็บไซต์เสมอ (ของคุณ) และจะไม่สามารถลบทิ้งได้ แต่สามารถแก้ไขรหัสผ่านได้ (ก็เป็นของคุณนี่)

การใช้งาน comment

  1. Log in เข้าสู่หน้า site-admin
  2. เมาส์ชี้ที่ส่วนเสริม > ความคิดเห็น > จัดการความคิดเห็น
  3. หากยังไม่ได้ติดตั้ง เช่นมี error doesn't exists ให้ทำการติดตั้งเสียก่อน จากนั้นระบบจะทำงานอัตโนมัติ และใช้งานได้ตามปกติ

การออกแบบธีม

  1. เปิด folder ไปที่ client/themes/ ตั้งชื่อธีมใหม่ เป็น folder ที่ตรงนี้
  2. copy ไฟล์จากในธีม default ไปวางไว้ใน folder ใหม่
  3. แก้ไขดีไซน์หรือออกแบบใหม่ตามใจชอบ โดยควรจะต้องมีตัวแทนแสดงผลที่เป็น <?php ... ?> เดิมอยู่ในนั้น
  4. เมื่อทำไฟล์ธีมต่างๆเสร็จแล้ว ให้กำหนดชื่อธีมที่จะใช้ใน config
    1. เปิดไฟล์ application/config/website.php
    2. $config["usetheme"] = "default"; เปลี่ยนจาก default เป็นชื่อธีมใหม่ที่คุณสร้างขึ้น แล้วบันทึกไฟล์
    3. เสร็จขั้นตอน

คุณสามารถแปลภาษา, เปลี่ยนคำต่างๆบนเว็บไซต์ได้ โดยไปที่ application/language/thai แล้วแก้ไขข้อมูลต่างๆในไฟล์เหล่านั้น.

การใช้งานจริง

  1. เข้าไปเปลี่ยนข้อมูลอีเมลของ admin ใน phpmyadmin (ทำแค่ครั้งแรกเริ่มใช้งานครั้งเดียว ต่อไปทำผ่านหน้า admin ได้ตามปกติ)
  2. เมื่อเปลี่ยนอีเมลแล้ว ทำการ log in เข้าหน้า admin แล้วแก้ไขข้อมูลของ admin เพื่อตรวจดูว่าอีเมลเปลี่ยนไปเรียบร้อยแล้วหรือยัง ซึ่งปกติจะต้องเปลี่ยนเรียบร้อยแล้ว
  3. ทำการเปลี่ยนรหัสผ่าน จากเดิมที่ใช้รหัสผ่าน pass ไปเป็นอย่างอื่นแล้วแต่จะตั้ง
  4. Log out และ Log in อีกครั้งด้วยรหัสผ่านใหม่
  5. หาก Log in สำเร็จในขั้นตอนนี้ถือว่าเริ่มใช้งานจริงแล้ว หาก log in ไม่สำเร็จ ให้ลบตารางต่างๆในฐานข้อมูลทิ้ง แล้วเริ่มติดตั้งใหม่ตั้งแต่แรก เพราะคุณอาจพลาดขั้นตอนใดขั้นตอนหนึ่งไป
